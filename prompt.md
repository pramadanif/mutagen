# PHASE 2 — BACKEND (Off-Chain Services)

## 2.1 IBC Volume Relayer

**Tech:** Node.js with `@cosmjs/stargate`

**Implementation:**
```
1. Query Cosmos Hub RPC for IBC transfer events via tx_search
   - Endpoint: confirm via cosmos.directory (e.g., rpc.cosmos.directory/cosmoshub)
   - Query: message.module='ibc_transfer' over rolling 1-hour window
2. Compute volume delta: (current_window - previous_window) / previous_window
3. Sign and submit UpdateIbcVolume { delta } to the contract using relayer wallet
4. Run every 5 minutes via setInterval
5. Log every submission: timestamp, computed delta, tx hash, success/failure
6. HTTP health endpoint at /health returning { lastUpdate, status }
```

**Error handling:** Retry 3x with exponential backoff. If all retries fail, log error and continue next cycle. Never crash the process.

## 2.2 Regime Classifier (AI Component 1)

**Implementation — Rule-based scorer (deterministic, no ML training needed):**
```
Input: 3 Hub feature deltas (normalized 0-1 each)
  - bonded_ratio_delta (from x/staking native query)
  - gov_activity_delta (from x/gov native query: # active proposals + vote skew)
  - ibc_volume_delta (from relayer above)

Score calculation:
  - Each feature: abs(delta) * 33.33, capped at 33
  - regime_score = sum of all three, capped at 100
  - 0-30: CALM (tight distribution, safe pulls)
  - 31-60: ELEVATED (wider distribution, moderate risk)
  - 61-100: TURBULENT (extreme distribution, high risk/reward)

Fires ONLY when a fresh oracle update lands, NOT continuously.
Logs every inference: { timestamp, features, score, regime_label }
```

**Upgrade path:** If time allows, replace with a small scikit-learn model trained on historical Hub data. But the rule-based version is fully functional and auditable — which is what the judges want.

## 2.3 Auditor (AI Component 2)

**Implementation — can run inside the relayer process:**
```
Trigger: every K pulls (K = 10 for testnet demo)

Zero-Sum Index calculation:
  - Fetch last K pull payouts from contract
  - Compute Gini coefficient: measures payout concentration
  - If Gini > 0.6 (threshold): intervene

Intervention (exactly ONE action, hard-capped):
  - Adjust curve fee by +0.005 (max cumulative: +0.05)
  - OR adjust payout cap by -2% (max cumulative: -15%)
  - Choose whichever brings Gini closer to 0.4

Log every check: { timestamp, pull_count, gini_value, threshold, action_taken_or_none, new_param }
Log format must be machine-readable (JSON) for the dashboard to consume.
```

```javascript
// deploy.js — run with: node deploy.js
const { SigningCosmWasmClient } = require("@cosmjs/cosmwasm-stargate");
const { DirectSecp256k1HdWallet } = require("@cosmjs/proto-signing");
const { GasPrice } = require("@cosmjs/stargate");
const fs = require("fs");

const RPC = "https://rpc.testnet.cosmos.directory/cosmoshub-testnet"; // confirm at build time
const MNEMONIC = process.env.MNEMONIC; // relayer wallet mnemonic

async function deploy() {
  const wallet = await DirectSecp256k1HdWallet.fromMnemonic(MNEMONIC, { prefix: "cosmos" });
  const [account] = await wallet.getAccounts();
  const client = await SigningCosmWasmClient.connectWithSigner(RPC, wallet, {
    gasPrice: GasPrice.fromString("0.025uatom"),
  });

  // Upload WASM
  const wasm = fs.readFileSync("./artifacts/mutagen.wasm");
  const uploadResult = await client.upload(account.address, wasm, "auto");
  console.log("Code ID:", uploadResult.codeId);

  // Instantiate
  const initMsg = { /* initial contract state */ };
  const instantiateResult = await client.instantiate(
    account.address, uploadResult.codeId, initMsg, "MUTAGEN", "auto"
  );
  console.log("Contract:", instantiateResult.contractAddress);
}
deploy();
```
